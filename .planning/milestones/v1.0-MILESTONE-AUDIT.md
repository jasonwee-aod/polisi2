## Tech Debt

### Phase 4: Fix source_url Chain — Schema/Code Divergence (2 items)

**Item 1 — `documents.source_url NOT NULL` not dropped**

Phase 4 correctly made `source_url` nullable at every Python and TypeScript layer (`models.py`, `retrieval.py`, `client.ts`). However, no SQL migration was created to drop the NOT NULL constraint from `public.documents.source_url`.

- **File:** `supabase/migrations/20260228_01_phase1_schema.sql:9` — `source_url text not null`
- **Impact:** Latent. In practice, the Phase 4 fix ensures `source_url` propagates through the upload→manifest→persist chain (non-null for all new docs). The existing NOT NULL constraint is protective, but the nullable Python types are asymmetric with the schema. Any document inserted with `source_url=None` would violate the constraint.
- **Fix:** `ALTER TABLE public.documents ALTER COLUMN source_url DROP NOT NULL;`
- **Affected requirements:** INDX-03 (risk only, currently satisfied)

**Item 2 — `citations.source_url NOT NULL` not dropped**

`CitationRecord.source_url: str | None = None` in `api/src/polisi_api/models.py:29` but `public.citations.source_url text not null` in schema. `repository.add_citations()` inserts `citation.source_url` which may be `None` if retrieved chunks have null `source_url`.

- **File:** `supabase/migrations/20260228_01_phase1_schema.sql:60` — `source_url text not null`
- **Impact:** Latent. Retrieved chunks come from `public.documents` rows which have non-null `source_url` (enforced by DB). But the code allows `None`, creating an asymmetry. If any chunk ever reaches the citation insertion step with `source_url=None`, `add_citations()` fails with a Postgres NOT NULL violation causing a 500 error on `POST /api/chat`.
- **Fix:** `ALTER TABLE public.citations ALTER COLUMN source_url DROP NOT NULL;`
- **Affected requirements:** API-02, API-04 (risk only, currently satisfied)
