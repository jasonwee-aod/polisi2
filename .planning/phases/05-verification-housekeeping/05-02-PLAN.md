    **Part A — service.py (Anthropic 429):**

    Edit `api/src/polisi_api/chat/service.py`. In `AnthropicTextGenerator.generate()`, wrap the `await self._client.messages.create(...)` call with a try/except that catches `anthropic.RateLimitError` (import as `from anthropic import AsyncAnthropic, RateLimitError` — the AsyncAnthropic import is already there, just add RateLimitError) and returns a graceful fallback string instead of re-raising.

    The fallback string should be language-neutral:
    `"[Rate limit reached — please try again in a moment.]"`

    This string is returned as the `str` return value of `generate()`, so `ChatService.generate_reply()` will build a normal `AssistantResponse` from it. No 500 is raised.

    Do not catch other anthropic errors (APIConnectionError, APITimeoutError, etc.) — those should surface normally.

    Implementation:
    ```python
    async def generate(self, prompt: PromptPackage) -> str:
        try:
            response = await self._client.messages.create(
                model=self._model,
                max_tokens=1200,
                system=prompt.system,
                messages=[{"role": "user", "content": prompt.user}],
            )
            text_parts: list[str] = []
            for block in response.content:
                if getattr(block, "type", None) == "text":
                    text_parts.append(block.text)
            return "".join(text_parts).strip()
        except RateLimitError:
            return "[Rate limit reached — please try again in a moment.]"
    ```

    **Part B — retrieval.py (OpenAI 429):**

    Edit `api/src/polisi_api/chat/retrieval.py`. In `OpenAIEmbeddingClient.embed()`, after `response.raise_for_status()` (currently line 47), wrap the entire HTTP call block in a try/except that catches `httpx.HTTPStatusError` where `exc.response.status_code == 429` and returns an empty list `[]`.

    `httpx` is already imported at the top of retrieval.py — no new import needed.

    Returning `[]` from `embed()` means `PostgresRetriever.retrieve()` propagates the empty embedding to the DB call, which will return no rows, and `generate_reply()` in service.py will fall through to `build_no_information_text()` — a graceful degradation with no crash and no 500. Do not catch other httpx.HTTPStatusError codes (401, 500, etc.) — only 429.

    Implementation for `OpenAIEmbeddingClient.embed()`:
    ```python
    async def embed(self, text: str) -> list[float]:
        try:
            async with httpx.AsyncClient(timeout=30) as client:
                response = await client.post(
                    "https://api.openai.com/v1/embeddings",
                    headers={
                        "Authorization": f"Bearer {self._api_key}",
                        "Content-Type": "application/json",
                    },
                    json={"input": text, "model": self._model},
                )
                response.raise_for_status()
            payload = response.json()
            return [float(value) for value in payload["data"][0]["embedding"]]
        except httpx.HTTPStatusError as exc:
            if exc.response.status_code == 429:
                return []
            raise
    ```

    Note: the `async with` block is moved inside the `try` so the exception propagates correctly. The `payload = response.json()` line must move outside the `async with` block but inside the `try` — keep it consistent with existing placement (response is still accessible after `async with` exits since the context manager only closes the connection, not the response object). If `response.json()` is currently outside the `async with`, leave it there; wrap only the entire body in try/except.
