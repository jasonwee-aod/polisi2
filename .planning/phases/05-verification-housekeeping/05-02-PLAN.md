    Edit `api/src/polisi_api/chat/service.py`. In `AnthropicTextGenerator.generate()`, wrap the `await self._client.messages.create(...)` call with a try/except that catches `anthropic.RateLimitError` and returns a graceful fallback string instead of re-raising.

    The fallback string should be language-neutral (it is returned as the answer text, so it should be understandable in both BM and English context). Use:
    `"[Rate limit reached ã please try again in a moment.]"`

    This string is returned as the `str` return value of `generate()`, so `ChatService.generate_reply()` will build a normal `AssistantResponse` from it. The caller receives a valid response; the user sees the fallback message inline in the chat. No 500 is raised.

    Import: add `import anthropic` at the top of the file if not already present (the `AsyncAnthropic` import is already `from anthropic import AsyncAnthropic` ã you can use `anthropic.RateLimitError` or import `RateLimitError` directly: `from anthropic import AsyncAnthropic, RateLimitError`).

    Implementation:
    ```python
    async def generate(self, prompt: PromptPackage) -> str:
        try:
            response = await self._client.messages.create(
                model=self._model,
                max_tokens=1200,
                system=prompt.system,
                messages=[{"role": "user", "content": prompt.user}],
            )
            text_parts: list[str] = []
            for block in response.content:
                if getattr(block, "type", None) == "text":
                    text_parts.append(block.text)
            return "".join(text_parts).strip()
        except RateLimitError:
            return "[Rate limit reached ã please try again in a moment.]"
    ```

    Do not catch other anthropic errors (APIConnectionError, APITimeoutError, etc.) ã those should surface normally so they are visible in logs and can be debugged. Only 429 rate-limit errors degrade silently to a user-facing message.
