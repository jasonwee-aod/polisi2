Root cause: SpacesUploader.upload_bytes calls boto3 put_object without a Metadata dict, so source_url is never stored on the S3 object. The manifest reads obj.metadata.get("source_url") which returns None for every object, yielding PendingIndexItem.source_url=None on every item. The Supabase documents table has source_url NOT NULL, so every chunk INSERT fails.

Fix 1 — scraper/src/polisi_scraper/core/spaces.py:
  Add metadata parameter to upload_bytes:
    def upload_bytes(self, data: bytes, object_key: str, content_type: str | None = None, metadata: dict[str, str] | None = None) -> str:
  Pass it to put_object:
    kwargs: dict = {
        "Bucket": self._config.bucket,
        "Key": object_key,
        "Body": data,
        "ContentType": guessed,
    }
    if metadata:
        kwargs["Metadata"] = metadata
    client.put_object(**kwargs)
  S3 Metadata values must be strings. Do not pass None values inside the metadata dict.

Fix 2 — scraper/src/polisi_scraper/runner.py:
  In _run_single_adapter, change the upload call to pass source_url as metadata:
    source_url_meta: dict[str, str] = {}
    if record.source_url:
        source_url_meta = {"source_url": record.source_url}
    if not dry_run:
        uploader.upload_bytes(payload, storage_path, metadata=source_url_meta if source_url_meta else None)
  Documents with no source_url (edge case where adapter produced no URL) skip setting metadata — manifest will yield source_url=None which is acceptable per the decision (fallback degrades gracefully).

Fix 3 — scraper/tests/test_dedup_and_state.py:
  Update FakeUploader.upload_bytes signature to accept the new metadata keyword argument so tests continue to pass:
    def upload_bytes(self, data: bytes, object_key: str, content_type: str | None = None, metadata: dict[str, str] | None = None) -> str:
  No logic change needed — FakeUploader just records the object_key.
